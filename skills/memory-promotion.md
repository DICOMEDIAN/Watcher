# Skill: memory-promotion

## Purpose

Promote selected facts from Notion logs and daily summaries into durable
Hermes operating memory.

## Promotion Criteria

Promote only if the fact changes future behavior.

Good candidates:

- new agent identity or role boundary
- durable user preference
- Notion message protocol rule
- bridge behavior rule
- encoding failure and fix
- project-independent operational decision
- recurring workflow convention

Do not promote:

- ordinary conversation
- temporary debugging output
- one-off task instructions
- incomplete `!!` drafts
- full review bodies
- raw file contents

## Storage Targets

```text
Raw log: Notion
Daily summary: D:\AI_Hermes\daily\YYYY-MM-DD.md
Durable ledger: D:\AI_Hermes\memory\*.md
Hot memory: Hermes MEMORY.md / USER.md
External provider: optional later
```

## Hot Memory Rule

Hermes hot memory is small. Store only high-impact facts that must affect every
future session.
